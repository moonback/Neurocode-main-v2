# Add Feature

Add a new feature to the codebase following best practices and project conventions.

## Instructions

1. **Understand the requirements:**
   - Ask the user to describe the feature
   - Clarify expected behavior and edge cases
   - Understand any constraints or dependencies
   - Identify affected components or modules

2. **Design the implementation:**
   - Review existing code patterns and conventions
   - Decide where the feature should live in the codebase
   - Identify required changes to types, interfaces, or APIs
   - Consider backward compatibility
   - Plan for error handling

3. **Implement the feature:**
   - Create or modify necessary files
   - Follow existing code style and patterns
   - Add proper type annotations
   - Implement error handling
   - Add logging or debugging aids if appropriate
   - Keep changes focused and minimal

4. **Add tests:**
   - Write unit tests for new functions/methods
   - Add integration tests if needed
   - Test happy path and edge cases
   - Ensure tests are comprehensive

5. **Update documentation:**
   - Add JSDoc comments to new functions
   - Update README or other docs if needed
   - Add inline comments for complex logic
   - Update type definitions

6. **Verify the implementation:**
   - Run all tests to ensure nothing broke
   - Run linting and type-checking
   - Manually test the feature
   - Check for performance implications

7. **Review and polish:**
   - Review your changes for quality
   - Remove any debug code or console logs
   - Ensure consistent naming and formatting
   - Verify all requirements are met
